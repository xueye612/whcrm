# CRM 项目介绍

## 1. 项目概述

本仓库是一个 CRM 二次开发项目，采用前后端分离模式：

- 前端工程：`crm_web-master`（Vue 2 + webpack）
- 后端工程：`crm_php-master`（ThinkPHP 5.0.24）
- 部署辅助：`deployment`（部署脚本与增量 SQL）

当前代码基线包含标准 CRM 能力，并扩展了收支管理（finance）和客户台账（ledger）模块。

## 2. 技术栈

### 后端

- PHP >= 7.0（项目资料中建议使用 7.3）
- ThinkPHP 5.0.24
- Composer 依赖包含：队列、迁移、Worker、PHPExcel/PhpSpreadsheet、PHPWord 等
- 缓存默认配置为 Redis

### 前端

- Vue 2.5.17
- Vue Router 3.0.1
- Vuex 3.0.1
- Element UI 2.12.0
- Axios 0.18.0
- webpack 4 构建

## 4. 后端模块与路由

后端启用了多模块架构，允许访问模块如下：

- admin
- crm
- oa
- bi
- work
- finance
- ledger

路由按模块拆分在 `crm_php-master/config` 目录下，主配置会加载：

- `route_admin.php`
- `route_crm.php`
- `route_oa.php`
- `route_bi.php`
- `route_work.php`
- `route_finance.php`
- `route_ledger.php`

### 扩展模块（重点）

1) finance（收支管理）

- 收支类型：`finance/type/*`
- 收支流水：`finance/record/*`
- 收支计划：`finance/plan/*`
- 支付方式：`finance/paymentmethod/*`

2) ledger（客户台账）

- 台账列表/详情/新增/更新/删除：`ledger/*`
- 导出：`ledger/excelExport`
- 台账客户列表：`ledger/customer/list`
- 台账记录：`ledger/record/list`、`ledger/record/add`

## 5. 前端结构与联调方式

前端核心目录：

- `src/api`：接口封装
- `src/router`：路由配置
- `src/views`：页面实现
- `src/store`：状态管理

开发环境默认端口为 `8090`，并通过代理转发：

- `/api` -> `目标地址/index.php`
- `/file` -> `目标地址/`

可通过环境变量 `PROXY_TARGET` 覆盖默认代理目标。

## 6. 本地运行指南

### 前端

建议先切换 Node 16（Windows + nvm）：

```bash
nvm use 16
node -v
npm -v
```

```bash
cd crm_web-master
npm install --legacy-peer-deps
npm run dev
```

默认访问地址：`http://localhost:8090`

### 后端

建议环境：Nginx + PHP-FPM + MySQL + Redis

基础步骤：

1. 将 `crm_php-master` 部署到 Web 目录
2. 配置 `crm_php-master/config/database.php`
3. 导入基础 SQL（以项目实际数据库脚本为准）
4. 确认 Web 服务用户对 `runtime`、`public` 具备写权限

## 7. 部署与数据库变更

### 自动部署脚本

`deployment/deploy.sh` 用于后端包部署，流程包括：

- 自动备份旧版本
- 解压并覆盖新版本
- 保留/恢复数据库配置
- 设置目录权限
- 重启 Nginx 与 PHP-FPM

### 增量 SQL 示例

`deployment/sql/20260205_add_feedback_time_to_customer_ledger.sql` 用于给客户台账表添加 `feedback_time` 字段，脚本具备幂等检查（字段存在则跳过）。

## 8. 开发与维护建议

1. 新增业务优先落在对应模块目录（controller/logic/model），避免跨模块耦合。
2. 新增后端模块时，同步更新模块白名单与路由配置加载项。
3. 前端新增页面建议按模块放在 `src/views`，并在 `src/router/modules` 注册。
4. 涉及 finance、ledger 的功能变更，需同步更新 SQL 脚本和接口文档。
5. 生产环境请关闭调试配置（如 `app_debug`、`app_trace`），并替换默认缓存/数据库敏感配置。
